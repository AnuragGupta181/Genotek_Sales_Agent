QA/QC Failure Report — Why Assumptions Weren't Caught
Root cause:
The previous research system had 8 scheduled agents running for 24 hours, but NO checker agent. No agent was tasked with asking "is this verified or assumed?" Every agent was additive — finding more details, synthesizing more data — but none was adversarial. The self-learning coordinator tracked coverage percentage but never questioned whether coverage meant accuracy.
What went wrong, specifically:

What should have happened
1
Name: "Autonomous Sales Manager"
— implies replacement, not support. Bijoy said "sales support agent" explicitly.
A checker should have compared the deliverable name against Bijoy's original request verbatim.
2
16-week timeline
— generated from a generic enterprise playbook. Bijoy builds in 2-day sprints with one person.
Timeline should have been derived from Bijoy's stated build cadence, not industry defaults.
3
No tool stack analysis
— zero comparison of OpenClaw, AgentScope, Claude native, or cost models.
Before ANY implementation plan, the first deliverable should have been "what do we build WITH."
4
Nate insights collected but not applied
— 16 insights mapped to workflow steps, but the mapping was cosmetic. The actual flowchart didn't change its architecture.
Each Nate insight should have produced a concrete design decision with a before/after.
5
No adversarial checker
— 8 research agents, 0 QA agents. Every agent was incentivized to ADD information, none to CHALLENGE it.
At least 1 agent should have run hourly with the sole job of marking claims as VERIFIED / ASSUMED / CONTRADICTED.
6
Claude leak not researched
— Chaofan Shou's discovery (March 31, 2026 — literally yesterday) exposed Claude Code's full architecture. Not a single agent checked current events.
